# Getting Started with Veeam Data Cloud for Microsoft Azure


To quickly get started with Veeam Data Cloud for Microsoft Azure, follow these steps:

1. Obtain access to Veeam Data Cloud and perform your first login. For details, see [Accessing Veeam Data Cloud](accessing_vdc.md).
2. Invite other users to your Veeam Data Cloud organization and assign them roles so that your team can use the portal. For details, see [Users and Roles](users_manage.md).
3. Learn about Veeam Data Cloud for Microsoft Azure limitations and find out in which regions you can store your backups. For details, see [Considerations and Limitations](azure_limitations.md) and [Backup Storage Regions](azure_regions.md).
4. Add your Microsoft Azure tenants to Veeam Data Cloud. For details, see [Adding Microsoft Azure Tenants](azure_tenant_add.md).
5. After you add a Microsoft Azure tenant, you can create backup policy to protect your tenant resources. For details, see [Microsoft Azure Backup](azure_backup.md).

|  |
| --- |
| Note |
| It is normal for the initial full backup to take a long time to complete. |

1. Once Veeam Data Cloud creates the first backup, you can restore your tenant data. For details, see [Microsoft Azure Restore](azure_restore.md).
2. View backup and restore logs and monitor the protection status of your Microsoft Azure tenant. For details see [Viewing Logs](azure_logs.md) and [Viewing Dashboard](azure_dashboard.md).

If you have other workloads in Veeam Data Cloud, you can leverage the [Organization Dashboard](dashboard.md) and [Organization Activity](activity.md) views, which provide aggregated information across your entire Veeam Data Cloud organization.
