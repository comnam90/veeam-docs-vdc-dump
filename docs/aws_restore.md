# Performing Restore


In various disaster recovery scenarios, you can perform the following restore operations using backed-up data:

* [Restore of EC2 instances](aws_restore_ec2.md) — restore EC2 instances and volumes from cloud-native snapshots or image-level backups to the original location or to a new location.
* [Restore of RDS resources](aws_restore_rds.md) — restore DB instances and Aurora DB clusters (from cloud-native snapshots) and DB instance databases (from image-level backups) to the original location or to a new location.
* [Restore of DynamoDB tables](aws_dynamo_restore.md) — restore DynamoDB tables from cloud-native backups to the original location or to a new location.
* [Restore of Redshift clusters](aws_redshift_restore.md) — restore Redshift clusters from cloud-native backups to their original location.
* [Restore of Redshift Serverless namespaces](aws_redshift_serverless_restore.md) — restore Redshift Serverless namespaces from cloud-native backups to the original, any existing or a new namespace.
* [Restore of EFS file systems](aws_efs_restore.md) — restore file systems from cloud-native backups to the original location or to a new location.
* [Restore of FSx file systems](aws_fsx_restore.md) — restore file systems from cloud-native backups to the original location or to a new location.
