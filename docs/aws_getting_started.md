# Getting Started with Veeam Data Cloud for AWS


To start working with Veeam Data Cloud for AWS, follow these steps:

1. Obtain access to Veeam Data Cloud and sign in to the portal. For details, see [Accessing Veeam Data Cloud](accessing_vdc.md).
2. [Optional] Invite additional users to your Veeam Data Cloud organization and assign roles to these users to let them access the portal. For details, see [Users and Roles](users_manage.md).
3. Check limitations and prerequisites for Veeam Data Cloud for AWS. For details, see [Considerations and Limitations](aws_limitations.md).
4. Create tenants to access AWS services and resources. For details, see [Creating Tenants](aws_tenant_add.md).
5. [Optional] Add database accounts to access databases of PostgreSQL DB instances. For details, see [Adding Database Accounts](aws_settings_adding_database_accounts.md).
6. Create backup policies to protect your AWS resources. For details, see [Creating Backup Policies](aws_backup_policies.md).
7. Perform restore operations to recover the backed-up AWS resources. For details, see [Performing Restore](aws_restore.md).
8. Monitor backup performance and view the results of data protection operations for specific tenants. For details, see [Viewing Dashboards](aws_dashboard.md#administration_dashboard).

If you have other workloads in Veeam Data Cloud, you can use the Organization Dashboard that allows you to estimate the overall health state of your entire Veeam Data Cloud organization. For details, see [Organization Dashboard](dashboard.md).

1. Track statistics of all running and completed operations, and view user activity in Veeam Data Cloud for AWS. For details, see [Viewing Logs](aws_logs.md).

If you have other workloads in Veeam Data Cloud, you can use the Activity View that allows you to monitor statistics and user activity across your entire Veeam Data Cloud organization. For details, see [Organization Activity](activity.md).
